# Split compile machine vs deploy machine

Use one environment to **compile Solidity** and commit artifacts, and another to **deploy** with only Node/Python + `.env` (no `hardhat compile` required on the deploy host).

## What gets committed

| Path | Purpose |
|------|---------|
| `artifacts/contracts/StakeWrap.sol/StakeWrap.json` | Hardhat artifact: **ABI**, **`bytecode`** (constructor), metadata. Used by `scripts/deploy.py` and `evm/contract.py`. |
| `abi/StakeWrap.abi.json` | ABI only (readable / tooling). Regenerated by `npm run compile`. |

Other `artifacts/` and `cache/` stay **gitignored**.

## Compile machine (build server)

1. Change contracts under `contracts/` as needed.
2. From repo root:
   ```bash
   npm ci
   npm run compile
   ```
   This runs `hardhat compile` and refreshes `abi/StakeWrap.abi.json`.
3. Commit and push:
   ```bash
   git add artifacts/contracts/StakeWrap.sol/StakeWrap.json abi/StakeWrap.abi.json contracts/
   git commit -m "chore: refresh StakeWrap build artifacts"
   git push
   ```

## Deploy machine

1. Clone or `git pull` (must include `StakeWrap.json` above).
2. Python venv + deps (e.g. `pip install -r requirements.txt` if you use one).
3. Configure `.env`: `PRIVATE_KEY`, `RPC_URL`, and any `bt_utils` / app env vars your `deploy.py` path needs (delegates, delegate coldkey passwords, etc.). `deploy.py` computes the base fees on-chain from the delegate coldkeys, so their passwords (`DELEGATE_1_PASSWORD`, `DELEGATE_2_PASSWORD`) must be available.
4. From repo root:
   ```bash
   python3 scripts/deploy.py
   ```
   **You do not need** `npm run compile` on this host if the artifact is already in git.

Optional: install only production deps if you never compile here; `deploy.py` only reads JSON via Python.

## After deploy

- `deployment.json` is **gitignored** (local address + ABI copy). Back it up or store in your secrets store.
- Run proxy / `setExecutor` steps as in your usual ops docs.

## Version skew warning

The committed `StakeWrap.json` must match the **same revision** as `contracts/*.sol` in that commit. If you edit Solidity without recompiling and pushing the artifact, deploy will ship **old bytecode**.
