🔐 StakeWrap Control

Stake

Add stake (market) or stake limit order. Uncheck "Not limited" for limit order.
Amount will be converted to rao (1 TAO = 10^9 rao)
Ignored when "Use Minimum Tolerance" is checked

Unstake

Remove stake (market) or unstake limit order. Amount in raw alpha units (integer). Leave amount empty to unstake all.
Ignored when "Use Minimum Tolerance" is checked

Transfer Stake

Transfer stake to the predefined allowed coldkey only. Leave amount empty to transfer the full stake on the origin netuid.

Move Stake

Move stake between different hotkeys. Leave amount empty to move all (alpha).

Withdraw TAO

Withdraw TAO from the contract (owner only). Leave amount empty to withdraw the full contract balance.

Fast Stake

Stake via MevShield (Bittensor extrinsic). Requires Executor ON (Settings). Uncheck "Not limited" for limit order.
Ignored when "Use Minimum Tolerance" is checked

Fast Unstake

Unstake via MevShield (Bittensor extrinsic). Requires Executor ON (Settings). Unstakes all for the given netuid.

Settings

Executor / heartbeat

Single switch in executor_enabled.json. When OFF, auto-execute scripts skip submissions.

Default hotkey

Hotkey (SS58) that execute() stakes/unstakes on. Owner only; the signer must be the contract owner.

Contract config

Read-only values from the deployed StakeWrap contract.
Load Settings to see contract config.

Tolerance offset

Used when "Use Minimum Tolerance" is checked (stake/unstake limit). Number (e.g. 0.001) or multiplier *1.2.

Stake Info

Click Refresh above to load stake info.
Subnet ID (click to set Stake / Unstake / Fast Stake / Fast Unstake)

Special operations

One-click actions that bypass the Stake tab fields. Same auth and contract rules as normal staking. EVM StakeWrap stake-limit. Uses the contract coldkey balance (same basis as server compute_contract_stake_all_amount_rao), netuid 28, default hotkey, minimum tolerance, no partial fill. MevShield / Bittensor extrinsic (same encoded amount as above). Requires Executor ON and contract staking enabled; delegates must have funds for execute().
